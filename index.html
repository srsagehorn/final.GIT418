JavaScript Classes Tutorial

In ES6, released in 2015, JavaScript introduced a new feature called classes. This allows developers to create similar objects easily by creating a basic format for each class and then instantiating and adjusting the class from there. Its important to note that even though classes may look like objects, they are not, they are simply templates for objects.

Why use classes?

Say for example that you wanted to create the following objects in JS you would do it like this:

Syntax

You instantiate every class with the word class followed by the name and a set of curly brackets. Typically the name is capitalized (unlike most other variable) because it signals to others that this is a class and not a standard object. Using the first example, lets call this class Cars. 

class Cars {
              }

Next comes the constructors. These are the things that will be consistent across all versions of objects this class will create. Each constructor will be inside of parentheses following the word constructor. Each one is separated by a comma with curly brackets after the parentheses. For the above example, it would be make, model, color, and year. 

class Cars {
        constructors(make, model, color, year) {
                }
              }

Finally, you will assign each constructor a value within the object. This may seem redundant but it lets the object know which key and value pairs go together. Each on will start with a this.[constructor] = [constructor]. 

class Cars {
        constructors(make, model, color, year) {
              this.make = make;
              this.model = model;
              this.color  = color;
              this.year = year;
                }
              }

So now that we have that class created what can we do with it? Well now we can simplify our example by creating each object.
